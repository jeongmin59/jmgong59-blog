## 텍스트 서식

일반 문단은 이렇게 작성됩니다. 긴 글을 쓸 때 줄 간격과 글자 간격이 얼마나 읽기 좋게 설정되어 있는지 확인할 수 있습니다. 한국어와 영어를 혼용할 때도 자연스럽게 흐릅니다.

**굵은 텍스트**는 중요한 내용을 강조할 때 사용합니다. *기울임 텍스트*는 용어나 외래어 표기에 씁니다. ~~취소선~~ 은 더 이상 유효하지 않은 내용에 사용합니다. `인라인 코드`는 변수명이나 짧은 명령어에 씁니다.

**굵게** + *기울임*을 함께 쓰면 **_이렇게_** 됩니다.

링크는 [이런 형태](https://posthog.com)로 작성하고, 외부 링크에는 별도 처리를 하지 않습니다.

## 헤딩 계층

헤딩은 `##` (H2) 와 `###` (H3) 두 단계를 지원합니다. H1은 포스트 제목으로 자동 사용됩니다.

### H3 — 소제목 예시

H3는 H2 섹션 내부의 세부 항목을 나눌 때 씁니다. 목차(TOC)에도 들여써서 표시됩니다.

### 또 다른 H3

여러 개의 H3을 연속으로 사용해 섹션을 세분화할 수 있습니다.

## H2 — 섹션 제목 예시

H2는 큰 문맥 전환이 있을 때 사용합니다. 목차에서 최상위 레벨로 보이며, 본문에서도 구분선과 함께 강조됩니다.

### H3 — H2 내부 소제목 예시

H3는 지금처럼 해당 H2 아래의 세부 항목을 구성할 때 사용합니다.

## 코드 블록

### TypeScript

```ts
// 모든 포스트를 날짜 내림차순으로 반환
export function getAllPosts(): PostMeta[] {
  return ensurePostsDirectory()
    .map((file) => {
      const slug = file.replace(/\.mdx$/, "");
      const { frontmatter, content } = parsePostFile(slug);
      return {
        ...frontmatter,
        slug,
        readingTime: readingTime(content).text,
        resolvedThumbnail: resolveThumbnail(frontmatter.thumbnail),
      } satisfies PostMeta;
    })
    .filter((post) => !post.draft)
    .sort((a, b) => +new Date(b.date) - +new Date(a.date));
}
```

  return (

      </div>

}
